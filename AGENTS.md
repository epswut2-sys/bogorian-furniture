## Project overview

**Bogorian Furniture** — Rak dinding & furnitur minimalis langsung dari pengrajin lokal

Bogorian Furniture is an Indonesian home decor marketplace connecting local craftsmen with buyers seeking quality wall iron shelves and minimalist furniture. Browse, add to cart, and checkout in minutes — no account needed.

Generated by Skylia AI Startup Builder. Full-stack Next.js SaaS application.

## Tech stack

  - auth: Clerk
  - email: Resend
  - hosting: Vercel
  - database: Neon
  - payments: Stripe
  - analytics: PostHog
  - monitoring: Sentry
  - Framework: Next.js 14 (App Router) + TypeScript
  - Styling: Tailwind CSS + shadcn/ui
  - Deployment: Vercel

## Directory structure

```
src/
  app/              # Next.js App Router pages and API routes
    api/            # API route handlers
    dashboard/      # Authenticated dashboard pages
  components/
    ui/             # shadcn/ui components (Button, Input, Card, etc.)
  lib/              # Shared utilities (db client, stripe, email, etc.)
prisma/             # Database schema and migrations
public/             # Static assets
```

## Key files

- `src/app/layout.tsx` — Root layout with providers (auth, theme)
- `src/app/page.tsx` — Landing page
- `src/app/dashboard/page.tsx` — Main authenticated dashboard
- \`middleware.ts\` — Clerk auth middleware (protects /dashboard routes)
- \`src/app/api/webhooks/stripe/route.ts\` — Stripe webhook handler
- \`prisma/schema.prisma\` — Database schema
- \`src/lib/db.ts\` — Prisma client singleton
- `.env.example` — Required environment variables

## Conventions

- Use `cn()` from `src/lib/utils.ts` for conditional classNames
- All UI components are in `src/components/ui/` (shadcn/ui pattern)
- API routes use Next.js Route Handlers (`route.ts` files)
- Protected pages check auth in server components or middleware
- Environment variables are typed in `.env.example`

## Common tasks

- **Add a new page**: Create `src/app/<path>/page.tsx`
- **Add an API route**: Create `src/app/api/<path>/route.ts`
- **Add a UI component**: Use shadcn/ui CLI or create in `src/components/ui/`
- **Modify database**: Edit `prisma/schema.prisma`, then run `npm run db:push`
- **Add env variable**: Add to both `.env.example` and `.env.local`
